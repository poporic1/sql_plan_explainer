# SQL Plan Explainer

## Описание

Программа берет SQL-запросы из файлов, получает план выполнения и делает по нему простой отчет:

- какие операции используются
- какие таблицы участвуют
- какая стоимость выполнения
- примерная оценка сложности
- рекомендации, на что обратить внимание

Сделано для того, чтобы быстрее понимать, что происходит внутри запроса, не разбирая EXPLAIN вручную.

---

## Как это работает

1. В папке `queries/` лежат SQL-файлы
2. Программа берет каждый файл и отправляет запрос в PostgreSQL через EXPLAIN
3. Полученный план разбирается:

   - ищутся операции (Seq Scan, Hash Join и т.д.)
   - вытаскиваются таблицы
   - находится cost
4. На основе этого собирается отчет
5. Отчет сохраняется в папку `output/`

## Как запустить

1. Убедись, что PostgreSQL запущен
2. Проверь настройки подключения в `db_runner.py`:

```
host = 'localhost'
port = 5432
dbname = 'mydb'
user = 'postgres'
password = '1234'
```

3. Положи SQL-файлы в папку `queries/`

4. Запусти:

```
python main.py
```

После этого в папке `output/` появятся отчеты.

---

## Что будет в отчете

Для каждого запроса создается файл вида:

```
query_01_summary.md
```

Внутри:

- список операций
- краткое объяснение
- таблицы
- cost
- оценка (low / medium / high)
- рекомендации

---

## Зачем это вообще нужно

Когда смотришь EXPLAIN, он часто выглядит перегруженно.
Этот скрипт помогает быстро понять:

- есть ли полный скан таблицы
- используются ли индексы
- есть ли тяжелые join'ы
